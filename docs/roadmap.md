# 飞鹊 (Feique) Roadmap

> 团队 AI 编程协作中枢 — Team AI Coding Orchestration Hub

## 路线图总览

| Phase | 版本 | 主题 | 目标时间 |
|-------|------|------|----------|
| 1 | v0.2 | 可观测性 + 成本感知 | 2026 Q2 |
| 2 | v0.3 | 组织级权限 + 审批 | 2026 Q3 |
| 3 | v0.4 | 治理与合规 | 2026 Q3–Q4 |
| 4 | v0.5 | 部署工业化 | 2026 Q4 |
| 5 | v0.6+ | 开放生态 | 2027+ |

---

## Phase 1: 可观测性 + 成本感知 (v0.2)

**核心目标**: 让团队管理者看到 AI 编程的使用状况和成本分布。

| 功能 | 说明 |
|------|------|
| Web 仪表板 | 嵌入式 Web UI，展示项目、会话、排队和后端运行状态 |
| 成本追踪 | 按项目/用户统计 token 用量和预估成本，支持告警阈值 |
| 每日摘要 | 自动生成团队 AI 编程日报，推送到指定飞书群 |
| 指标增强 | 扩展 Prometheus 指标：响应延迟分位数、后端错误分类、队列深度趋势 |

---

## Phase 2: 组织级权限 + 审批 (v0.3)

**核心目标**: 支持部门级管理和敏感操作审批。

| 功能 | 说明 |
|------|------|
| 部门 RBAC | 基于飞书组织架构的权限继承，支持部门级项目可见性控制 |
| 审批流 | 敏感操作（如生产仓库写入、配置变更）接入飞书审批流程 |
| 多频道通知 | 操作结果按项目/角色推送到不同飞书群，避免信息噪音 |
| 配额管理 | 按部门/项目设置每日 token 额度和并发上限 |

---

## Phase 3: 治理与合规 (v0.4)

**核心目标**: 满足企业合规审计要求。

| 功能 | 说明 |
|------|------|
| 合规仪表板 | 展示权限覆盖率、审计完整性和策略合规状态 |
| 审计增强 | 完整操作日志链、数据留存策略、导出功能 |
| 会话回放 | 可视化回放 AI 编程会话的完整交互过程 |
| 策略引擎 | 可配置的操作策略（如禁止特定命令、限制工作时间） |

---

## Phase 4: 部署工业化 (v0.5)

**核心目标**: 生产级部署和运维工具链。

| 功能 | 说明 |
|------|------|
| Docker / K8s | 官方 Docker 镜像和 Helm Chart，支持一键部署 |
| 高可用 | 多实例部署、状态同步、故障自动切换 |
| 运维工具 | 健康检查增强、平滑升级、配置热加载、备份恢复 |
| 监控集成 | 开箱即用的 Grafana 仪表板和告警规则模板 |

---

## Phase 5: 开放生态 (v0.6+)

**核心目标**: 开放能力，支持更广泛的场景。

| 功能 | 说明 |
|------|------|
| 插件系统 | 自定义命令、钩子和中间件的注册机制 |
| 更多后端 | 支持 Cursor、Windsurf 等更多 AI 编程工具 |
| 多租户 | 单实例服务多个团队，数据完全隔离 |
| API 开放 | RESTful API 供第三方系统集成 |

---

## 设计原则

1. **渐进增强**: 每个 Phase 独立交付价值，不阻塞后续功能
2. **向后兼容**: 配置和 API 升级不破坏现有部署
3. **本地优先**: 数据默认存储在用户控制的基础设施上
4. **可观测优先**: 新功能必须自带指标和审计日志
