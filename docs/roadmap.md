# 飞鹊 (Feique) Roadmap

> 团队 AI 协作中枢 — Team AI Collaboration Hub
>
> 从个人提效到团队协作，让 AI 融入工作的每个环节。

## 愿景

飞鹊的目标不限于 AI 编程——它要成为团队使用 AI 协作的基础设施。让团队成员协作配合、相互促进，帮助发现问题和瓶颈、提升效率，推动从个人能力的提升，到团队协作的顺畅，再到整体能力的持续迭代。

当前版本（v0.2）已实现 AI 编程场景的完整链路，并内置了团队协作、知识沉淀和可观测性能力。后续版本将逐步拓展到更多类型的工作和工作的更多环节。

## 路线图总览

| Phase | 版本 | 主题 | 目标时间 |
|-------|------|------|----------|
| 1 | v0.2 | 可观测性 + 成本感知 | ~~2026 Q2~~ **已发布** |
| 2 | v0.3 | 组织级权限 + 审批 | 2026 Q3 |
| 3 | v0.4 | 治理与合规 | 2026 Q3–Q4 |

## Phase 1: 可观测性 + 成本感知 (v0.2) ✅ 已完成

**核心目标**: 让团队管理者看到 AI 使用的全貌——谁在用、用了多少、效果如何。

| 功能 | 说明 | 状态 |
|------|------|------|
| Web 仪表板 | 嵌入式 Web UI，展示项目、会话、排队和后端运行状态 | ✅ 已完成 |
| 成本追踪 | 按项目/用户统计 token 用量和预估成本，支持告警阈值 | ✅ 已完成 |
| 每日摘要 | 自动生成团队 AI 协作日报，推送到指定飞书群 | ✅ 已完成 |
| 效率洞察 | 识别瓶颈环节：哪些项目排队频繁？哪些任务耗时异常？ | ✅ 已完成 |
| 指标增强 | 扩展 Prometheus 指标：响应延迟分位数、后端错误分类、队列深度趋势 | ✅ 已完成 |

---

## Phase 2: 组织级权限 + 审批 (v0.3) — 部分完成

**核心目标**: 支持部门级管理和敏感操作审批，让协作有序可控。

| 功能 | 说明 | 状态 |
|------|------|------|
| 部门 RBAC | 基于飞书组织架构的权限继承，支持部门级项目可见性控制 | 🔜 计划中 |
| 审批流 | 敏感操作（如生产仓库写入、配置变更）接入飞书审批流程 | ✅ 已完成 |
| 多频道通知 | 操作结果按项目/角色推送到不同飞书群，避免信息噪音 | ✅ 已完成 |
| 配额管理 | 按部门/项目设置每日 token 额度和并发上限 | ✅ 已完成（v0.2 成本追踪含基础配额） |

---

## Phase 3: 治理与合规 (v0.4)

**核心目标**: 满足企业合规审计要求，让协作过程可追溯。

| 功能 | 说明 |
|------|------|
| 合规仪表板 | 展示权限覆盖率、审计完整性和策略合规状态 |
| 审计增强 | 完整操作日志链、数据留存策略、导出功能 |
| 会话回放 | 可视化回放 AI 协作会话的完整交互过程 |
| 策略引擎 | 可配置的操作策略（如禁止特定命令、限制工作时间） |

---

## Phase 4: 部署工业化 (v0.5)

**核心目标**: 生产级部署和运维工具链。

| 功能 | 说明 |
|------|------|
| Docker / K8s | 官方 Docker 镜像和 Helm Chart，支持一键部署 |
| 高可用 | 多实例部署、状态同步、故障自动切换 |
| 运维工具 | 健康检查增强、平滑升级、配置热加载、备份恢复 |
| 监控集成 | 开箱即用的 Grafana 仪表板和告警规则模板 |

---

## Phase 5: 开放生态 + 多场景 (v0.6+)

**核心目标**: 超越 AI 编程，让飞鹊成为团队使用 AI 的通用协作基础设施。

| 功能 | 说明 |
|------|------|
| 插件系统 | 自定义命令、钩子和中间件的注册机制 |
| 更多后端 | 支持 Cursor、Windsurf 等更多 AI 工具 |
| 多场景拓展 | 从编程延伸到文档写作、数据分析、设计评审等团队工作场景 |
| 多租户 | 单实例服务多个团队，数据完全隔离 |
| API 开放 | RESTful API 供第三方系统集成 |

---

## 设计原则

1. **渐进增强**: 每个 Phase 独立交付价值，不阻塞后续功能
2. **向后兼容**: 配置和 API 升级不破坏现有部署
3. **本地优先**: 数据默认存储在用户控制的基础设施上
4. **可观测优先**: 新功能必须自带指标和审计日志
5. **场景驱动**: 先做好一个场景（AI 编程），再拓展到更多工作类型
