# Contributing

感谢你关注这个项目。

## 适合贡献的方向

- Bug 修复
- 飞书事件兼容性
- Codex CLI 版本兼容
- 文档完善
- 安装和运维脚本增强
- 指标、告警、观测面板改进

## 基本流程

1. Fork 仓库并新建分支
2. 保持改动聚焦，优先修根因
3. 补相关测试或最小验证
4. 提交前运行：

```bash
pnpm typecheck
pnpm test
pnpm build
```

5. 提交 PR，说明：

- 改了什么
- 为什么这样改
- 如何验证
- 可能影响的行为

## 提交建议

- 使用清晰的 commit message
- 不要把真实凭证、状态目录、日志或打包产物提交进仓库
- 如果改动会影响飞书交互，请附带复现步骤

## 文档改动

文档和官网也是代码的一部分。

- README 面向首次访问 GitHub 仓库的人
- `docs/` 面向实际部署和排障的人
- `website/` 面向对外展示和 GitHub Pages 落地页
