让飞书成为 Codex 的会话入口，而不是提示词垃圾桶。

在飞书里继续你的 Codex session，切项目，控运行，查状态。

这不是一个只会转发消息的 bot。它是面向真实工程环境的桥接层：支持项目路由、session 续接、原生回复、幂等去重、审计日志、指标导出和后台运维命令。

Transport 2 modes

`long-connection` for local-first, `webhook` for shared deployment.

Session model Project-aware

Runtime Recoverable

Supports cancel, timeout, stale/orphaned run recovery and pid-backed management.

WHY THIS EXISTS

飞书负责沟通，Codex 负责执行，桥接器负责把上下文和控制权保住。

Message In

飞书消息进入桥接器，先做去重、白名单过滤、群聊 `@mention` 判断和命令解析。

Route + Resume

根据当前聊天窗口选中的项目，自动映射到正确的 Codex session，决定新开还是续接。

Reply + Observe

用飞书原生 reply 回消息，同时写审计、更新运行态、暴露指标，保证排障和运维可控。

MODES

两种接入方式，各自解决不同问题。

Local-first

long-connection

  • 无需公网回调
  • 最适合个人开发和内网环境
  • 启动快，验证链路短
Production

webhook

  • 支持卡片回调和公共部署
  • 便于接反向代理、监控和 HTTPS
  • 适合团队共享服务
