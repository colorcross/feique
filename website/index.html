让团队 AI 编程 有序、可控、可观测。

飞鹊 (Feique) 是团队 AI 编程协作中枢。把飞书消息路由到 Codex CLI 或 Claude Code，项目绑定、会话续接、仓库串行、后端切换、权限分层——一个中枢管好团队协作、运营管控和可观测性。

How it works

消息进入后端之前，先经过路由、会话和并发控制。

消息先进入项目绑定层

先取当前 chat 绑定的项目；没绑定就回退默认项目，而不是让每条消息都重新猜仓库。

Transport Modes

两种接入模式，覆盖个人开发到团队生产。

推荐个人使用

通过官方 SDK WebSocket 长连接接入，不需要公网回调地址，本机开发和个人使用首选。

  • 接入快，配置简单

通过飞书事件订阅和卡片回调接入，适合团队共享服务和生产部署，功能最完整。

Capabilities

团队协作、运营管控、可观测性——三个维度的关键能力。

Project Routing

/project <alias> 会真正改写当前 chat 的默认项目

群里切一次，整群后续都落到同一个项目；私聊也会记住各自的当前项目。

Session Adoption

/session adopt latest|list|<thread_id>

飞书侧不再只能续桥接器自己保存的 thread，也能接管本机 Codex CLI 的原生 session。

Auto Adopt

切项目后可自动接上最近会话

开启 project_switch_auto_adopt_latest = true 后，项目切换会优先保留当前 chat 会话，否则自动接最近匹配的本地 session。

Runtime Discipline

queue key + project.root 双层串行

同项目 thread 不会乱写，不同群同时操作同一仓库也会被自动收口。

Wiki & Knowledge

飞书知识库完整读写接入

支持 /wiki 搜索、读取、创建、改名、复制、移动和成员管理；/kb search 搜索项目本地文档。

MCP Surface

既支持 stdio，也支持 HTTP/SSE MCP

运行 feique mcp，即可把项目列表、项目切换、会话接管和自然语言控制命令暴露给 OpenClaw 等客户端；远端接入支持多 token 轮换和 Bearer 鉴权。

Memory System

项目记忆与群共享记忆

支持 /memory save/search/pin/forget/restore，SQLite + FTS5 检索，可配置 TTL、置顶策略和后台定时清理。

Observability

审计日志、健康探针、Prometheus 指标

内置 /healthz、/readyz、/metrics，支持 Alertmanager 告警和 Grafana 可视化，所有数据本地可控。

Access Roles

角色之外，再补 session / run / config / service 能力名单

可以把项目查看、会话控制、运行执行、配置修改、服务观察和服务重启拆开授权，而不是所有 operator 权限打包放开。

Admin Control

管理员控制面与动态配置

通过 /admin 命令动态维护管理员、群聊白名单、项目配置，支持配置历史快照和一键回滚。

Multi-Backend

/backend codex|claude 会话级后端切换

通过 /backend 命令或自然语言"后端切换到 claude"随时切换当前项目的后端。优先级链：会话级覆盖 → 项目 backend 配置 → 全局 backend.default。项目路由、会话续接、排队和权限体系对两种后端统一生效。

Quick Start

从安装到真实联调，只需要沿着这条最短路径走一遍。

CLI install and bootstrap 
# 安装
npm install -g feique
feique init --mode global

# 检查
feique doctor
feique doctor --remote

# 启动
feique start
feique logs --follow
feique create-project repo-new /srv/codex/repo-new
feique mcp

# 项目管理
/projects
/project repo-a
/admin project create repo-new /srv/codex/repo-new

# 会话管理

/session list

Configuration

一份最小配置就能跑起来，按需逐步扩展。

~/.feique/config.toml 最小配置示例 
version = 1

[service]
default_project = "default"
reply_mode = "card"  # text | post | card

[backend]
default = "codex"  # codex | claude

[codex]
bin = "codex"
default_sandbox = "workspace-write"

[storage]
dir = "~/.feique/state"

[security]
allowed_project_roots = ["/srv/repos"]
admin_chat_ids = ["oc_admin_chat_1"]

[feishu]
app_id = "env:FEISHU_APP_ID"
app_secret = "env:FEISHU_APP_SECRET"
transport = "long-connection"

[projects.default]
root = "/srv/repos/repo-a"
session_scope = "chat"
快速启动

只需设置 FEISHU_APP_ID 和 FEISHU_APP_SECRET 两个环境变量即可启动。

回复模式选择

个人使用推荐 post（富文本），团队部署且需要卡片按钮时切换到 card。

多项目绑定

用 feique bind <alias> <path> 绑定多个仓库，飞书里用 /project <alias> 切换。

安全边界

allowed_project_roots 限制 Codex 可操作的仓库目录，生产环境请填明确路径。
